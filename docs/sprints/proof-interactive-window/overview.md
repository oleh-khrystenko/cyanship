# Sprint: Proof Interactive Window

> Інтерактивне вікно у секції Proof лендінгу, яке дозволяє відвідувачам "помацати" продукт не покидаючи сторінку: авторизуватись, побачити Stripe checkout, переглянути показники швидкості.

---

## Що робимо

Зараз у секції Proof (DogfoodingSection) зліва є текст і три кроки, а справа — порожній простір. Додаємо туди інтерактивне вікно, яке перемикається в залежності від обраного кроку.

### Три режими вікна

Кроки вже існують у секції як текст. Робимо їх клікабельними — при натисканні на крок вікно справа показує відповідний контент.

**1. Auth — "Create an account to test the auth flow."**

Мета: показати що авторизація працює, прямо на лендінгу.

Якщо користувач НЕ авторизований:
- Чекбокс згоди з Terms of Service
- Кнопка "Continue with Google"
- Поле для email + кнопка відправки magic link
- Це мінімальний набір — повторює наш реальний auth flow

Якщо користувач авторизований:
- Аватарка користувача
- Його email та ім'я
- Кнопка "Log out"

Що НЕ робимо в цьому вікні:
- Password login — занадто багато UI для маленького вікна
- Реєстрацію з паролем — те саме
- Будь-які складні edge cases (lockout, rate limit UI)

Якщо якийсь крок потребує повноцінного UI (наприклад email вже зареєстрований з паролем) — просто перенаправляємо на /auth/signin. Наша ціль — proof of concept, не повна заміна auth сторінки.

**2. Billing — "Click the Stripe checkout to see the payment integration."**

Мета: показати що платіжна інтеграція реальна.

Вигляд однаковий для всіх:
- Показуємо каталог планів/паків (дані вже є через GET /payments/catalog)
- Кнопки "Subscribe" / "Buy" біля кожного варіанту

Поведінка залежить від стану авторизації:
- Авторизований — кнопки працюють, відкривається Stripe Checkout
- НЕ авторизований — вікно автоматично перемикається на вкладку Auth з повідомленням: "Платіжна система доступна тільки авторизованим користувачам. Спочатку створіть акаунт."

Stripe працює в test mode — відвідувач може ввести тестову карту і пройти повний flow.

**3. Lighthouse — "Experience the load speed firsthand."**

Мета: візуальне підтвердження швидкості.

- Кругові діаграми (gauge/donut) з анімацією заповнення
- Чотири метрики: Performance, Accessibility, Best Practices, SEO
- Всі показують 100%
- Значення захардкоджені — це не live тест, а демонстрація
- Красива анімація при появі (count-up від 0 до 100)

## Мобільна версія

На мобільному вікно з'являється як Bottom Sheet (висувна панель знизу):

- Три кроки відображаються як кнопки або карточки — щоб було очевидно що по них можна натиснути
- При натисканні знизу висувається панель з відповідним контентом
- Панель можна закрити свайпом вниз або кнопкою
- Кожна кнопка-крок має іконку та короткий текст, щоб навіть нетехнічний користувач зрозумів що це інтерактивні елементи

---

## UX деталі

- При першому завантаженні секції вікно показує Auth (перший крок) — це найбільш зрозумілий стартовий стан
- Активний крок зліва візуально виділений (колір, підкреслення, або інший індикатор)
- Перемикання між кроками — плавна анімація (fade або slide)
- Вікно має фіксовану висоту щоб контент не "стрибав" при перемиканні
- На десктопі вікно sticky — скролиться разом з кроками зліва

---

## Що не входить у цей спрінт

- Password-based login у вікні Auth
- Реєстрація з паролем
- Управління підпискою (portal) з вікна
- Реальний Lighthouse тест (live audit)
- Адаптація auth flow під всі edge cases — для складних випадків редірект на /auth/signin
